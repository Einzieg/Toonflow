# 故事板辅助单图模式开发规划

## 核心定义

故事板不是把分镜图拼起来。

Toonflow 的正确流程应为：

```text
剧本 + 分镜面板文本 + 资产描述 + 目标视频时长
  -> Agent 根据剧情节拍自动判断故事板分割
  -> Agent 生成分镜头脚本
  -> 图片模型根据分镜头脚本生成单页故事板图片
  -> 视频模型读取故事板图片 + 分镜头脚本生成视频
```

概念边界：

- **分镜面板**：结构化镜头数据，来源于工作流阶段，包含 `o_storyboard.videoDesc/prompt/duration/track` 等。
- **分镜图**：某条分镜自己的视觉帧，通常对应 `o_storyboard.filePath`，不是故事板的必需输入。
- **分镜头脚本**：Agent 根据剧本和分镜面板生成的导演执行文本，包含镜号、时长、景别、运镜、构图、角色调度、台词和声音。
- **故事板图片**：由图片模型根据分镜头脚本绘制的单页故事板工作页，类似导演拍摄说明页，不是已有分镜图的拼贴。
- **故事板视频**：以故事板图片作为单图参考，并把分镜头脚本放入提示词后生成的视频。

参考视觉形态：类似导演故事板页，包含多行镜头卡片、小画面框、镜号、时长、景别、运镜、台词和画面说明。参考图：`https://cdn3.ldstatic.com/original/4X/6/e/e/6ee55ddf28f264bf0f46e7d96aa6729b1a8b7161.jpeg`

## 设计原则

- 不读取 `o_storyboard.filePath` 作为故事板图片的输入。
- 允许分镜图尚未生成时创建故事板。
- 故事板分割由 Agent 自动判断，不能让前端固定按“每页 N 个镜头”机械切分。
- 故事板生成必须先产出 `shotScript`，再产出图片。
- 故事板图片生成应尽量引用已有分镜图、角色/场景/道具资产图作为参考，用于锁定项目实际画风和关键视觉元素。
- 故事板图片存在随机失败或风格偏差，必须支持单张故事板重新生成。
- 故事板图片只是视觉辅助，视频提示词必须同时引用数据库中的 `shotScript`。
- 故事板辅助单图模式是 Toonflow 的业务组织模式，底层视频模型仍使用真实能力，例如 `singleImage`。
- 故事板视频结果与现有视频工作台轨道隔离，避免污染 `o_videoTrack`。

## 数据结构

`o_storyboardBoard` 保存故事板页和脚本：

```ts
id
projectId
scriptId
storyboardIds // JSON number[]
startIndex
endIndex
filePath // 图片模型生成的故事板页
thumbPath
layout // 兼容旧字段；新模式固定 script
ratio // auto / 16:9 / 9:16
itemsPerBoard // Agent 自动分割后的实际镜头数，只做记录，不作为前端手动参数
labelMode // 新模式固定 shotScript
shotScript // Agent 生成的分镜头脚本
imagePrompt // 用于生成故事板页的图片提示词
imageModel
targetDuration
sourceType // scriptShotSheet
sourceHash
state
errorReason
createTime
updateTime
```

`o_storyboardBoardVideo` 继续保存故事板视频业务记录：

```ts
id
boardId
projectId
scriptId
videoId // 关联 o_video
referenceMode // storyboardImage
model
prompt
duration
resolution
state
errorReason
createTime
updateTime
```

## 后端接口

### 创建故事板

`POST /production/storyboardBoard/create`

入参：

```ts
projectId
scriptId
storyboardIds
ratio?
targetDuration?
```

行为：

- 读取项目配置：图片模型、画风、导演手册、视频画幅。
- 读取剧本内容和分镜面板文本。
- 读取分镜关联资产描述，优先使用有效衍生资产描述。
- 调用 `universalAi` 根据剧情节拍、动作连续性、台词完整性、场景转换和目标视频时长输出连续分段计划。
- 调用 `universalAi` 生成 `shotScript`。
- 调用项目图片模型，根据 `shotScript` 生成单页故事板图片；如存在可用分镜图或关联资产图，应压缩后作为参考图传入。
- 保存图片、缩略图、`shotScript`、`imagePrompt`、`targetDuration`。
- 相同输入 `sourceHash` 已完成时复用已有故事板。

校验：

- `storyboardIds` 必须属于当前 `projectId/scriptId`。
- 每条分镜至少要有 `videoDesc` 或 `prompt`。
- 不要求每条分镜已有 `filePath`。
- 项目必须配置可用图片模型。

### 故事板视频

`POST /production/storyboardBoard/generateVideo`

行为：

- 读取 `board.filePath` 作为唯一单图参考。
- 读取 `board.shotScript` 写入视频提示词。
- 使用视频模型的 `singleImage` 能力。
- 生成结果写入 `o_video`，`o_storyboardBoardVideo.videoId` 关联该视频。

提示词重点：

- 明确故事板图片是分镜头脚本页，不是静态拼贴图。
- 按镜头编号顺序推进剧情。
- 按最终提交时长压缩或扩展节奏。
- 台词和字幕必须输出中文；无台词镜头不要生成口型。

## 前端交互

分镜面板中的故事板区域应提供：

- 生成故事板
- 重新生成故事板图片
- 预览故事板图片
- 故事板图生视频
- 预览视频
- 删除故事板

生成参数：

- 范围：全部 / 当前选择 / Sxx-Sxx
- 故事板比例：跟随项目 / 16:9 / 9:16
- 视频模型：只展示支持 `singleImage` 的模型
- 视频目标时长、分辨率、音频
- 分割策略：后端 Agent 自动判断；前端不提供固定“镜头/页”配置。

选择逻辑：

- 只要求分镜存在 `id`。
- 不应因为没有 `src/filePath` 而过滤掉分镜。
- 空范围提示应是“当前范围没有可用分镜文本”，不是“没有可用分镜图”。

展示逻辑：

- 列表展示故事板缩略图。
- 元信息展示 `分镜头脚本 · N 个镜头 · 目标 Xs · ratio`。
- 老记录仍可显示，但新记录的 `sourceType` 应为 `scriptShotSheet`。

## 生成策略

分镜头脚本输出格式：

```md
# 分镜头脚本
总时长：Xs
## 镜头 01 / S01 / Xs
- 画面内容：
- 景别：
- 运镜：
- 构图：
- 角色调度：
- 台词/字幕：
- 声音/情绪：
- 故事板画面：
```

故事板图片提示词要求：

- 单页故事板工作页。
- 多个镜头卡片或横向行。
- 每个镜头有小画面框和简短中文字段。
- 不要拼贴已有分镜图。
- 参考图只用于锁定画风、角色、服装、场景、道具和材质，不要直接粘贴到故事板页里。
- 不要做宣传海报、九宫格截图或单帧剧照。
- 文字简短可读，避免密集小字。

重新生成策略：

- 重新生成只针对当前故事板页，不重新做 Agent 分割。
- 优先复用已有 `shotScript`，保持镜头内容稳定。
- 重新生成成功后替换 `filePath/thumbPath` 并清理旧图片文件。
- 重新生成会清理该故事板下已有视频记录，避免新图继续显示旧视频结果。

## 验收标准

- 没有分镜图时也能生成故事板。
- 生成前会先写入或返回分镜头脚本。
- 故事板图片内容是脚本页，不是分镜图拼接。
- 视频生成提示词包含 `shotScript`。
- 前端不再把无图片分镜过滤掉。
- 删除故事板不会删除分镜面板、分镜图或资产。
- 旧故事板记录可继续展示，新故事板记录使用 `sourceType=scriptShotSheet`。
